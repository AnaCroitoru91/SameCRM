# SameCRM — GitHub File-by-File Upload Guide

## Overview

Because GitHub's web uploader loses nested folder structure and binary React build assets can't be pasted as text, we've restructured the deploy package so every file can be created via **GitHub → Add file → Create new file**.

The React build (CSS + JS) has been inlined into a single `server/public/index.html` — no separate `/assets/` folder needed.

## How to Upload `server/public/index.html`

This file is ~351KB and too large to paste in a text editor. Use one of these approaches:

### Option A: GitHub Desktop / git CLI (recommended)
1. Clone your repo locally
2. Copy `server/public/index.html` from the File System panel (download it)
3. Place it at `server/public/index.html` in your local clone
4. `git add server/public/index.html && git commit -m "feat: inline react build" && git push`

### Option B: GitHub web drag-and-drop
1. Go to your repo on GitHub
2. Navigate to or create the `server/public/` folder
3. Click **Add file → Upload files**
4. Drag the `index.html` file from your downloads
5. Commit

## Step-by-Step GitHub Web Interface

1. Go to `https://github.com/YOUR_USERNAME/YOUR_REPO`
2. Click **Add file → Create new file**
3. In the filename box, type `package.json` (or `server/index.js` etc — slashes create folders)
4. Paste the file content from this guide
5. Click **Commit new file**
6. Repeat for each file in the table above
7. For `server/public/index.html`: use drag-and-drop upload (see above)
